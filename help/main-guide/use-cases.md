# Mix Modelerのユースケース

Mix Modelerでは、次の主要なユースケースを有効にします。

## オムニチャネルのパフォーマンスの増分を把握

このユースケースは、ペイドチャネル、アーンドチャネル、オウンドチャネルをまたいで、マーケティングの影響を測定するのに役立ちます。

+++ 詳細

### 課題

これらのユースケースアドレスの課題は次のとおりです。

* サイロ化されたカスタマージャーニーデータ、シグナルの損失、ウォールドガーデンなどから得られる増分パフォーマンスを測定することが困難です。
* MMMとMTAを分離した手法によるインサイトの一貫性が欠け、結果に対する信頼が低下する。
* 成功に導くマーケティングチャネルや戦術についての理解が不十分。

### アプローチ

このユースケースのステップベースのアプローチは次のとおりです。

| 手順 | 詳細 |
|---|---|
| 取り込み | 共通スキーマの下でデータソースを特定し、取り込むことができます。<br/>Adobe AnalyticsまたはCustomer Journey Analyticsへの既存の投資を活用して、迅速な展開を実現。 |
| 設定 | 特定のビジネス目標に対するAIaaS （サービスとしてのAI）フレームワークを使用して、柔軟なモデルを設定します<br/>双方向の転移学習により、タッチポイントと概要レベルの間の一貫性を自動的に確保します。 |
| 分析 | マーケティング全体と個々のチャネル/サブチャネルのROIを明確に把握します。<br/>増分コンバージョンを最も促進するタッチポイントを明確に把握します。 |

{style="table-layout:auto"}


### 影響

この使用を適切に実装すると、次のような影響を与える可能性があります。

* 集約データ、顧客接点データ、外部要因を組み込んで、測定に関する詳細な全体像を把握します。
* サービスとしてのAI ソリューションを使用して、成功を促進するためのモデルを迅速に作成し、自信レベルでモデルの透明性を確保する。
* サマリーレベルとタッチポイントレベルの結果の調整により、戦略的および戦術的な意思決定の信頼を高めることができます。

+++


## ROIを最適化するマーケティングプランを構築する

このユースケースでは、ユーザーフレンドリーでAIを活用した最適化とシナリオプランニングツールを使用して、ROIを最大化します。

+++ 詳細

### 課題

これらのユースケースアドレスの課題は次のとおりです。

* 直感に頼るのではなく、効率性のカーブにもとづいて、データドリブン型のマーケティング投資計画を作成できます。
* 複数の地域、事業部門、チャネルをまたいで同時に成果を最大化。
* 長いイテレーションサイクルにより、多くの手作業が必要な複数の予算シナリオを作成および比較できます。


### アプローチ

このユースケースのステップベースのアプローチは次のとおりです。

| 手順 | 詳細 |
|---|---|
| 設定 | ビジネス目標に合わせて測定モデルを簡単にカスタマイズできます。<br/> チャネル、地域、セールスサイクル、遅延、社内外のビジネス要因など、コーディングは不要で、数回クリックしてパラメーターを定義できます。 |
| トレーニング | 設定されたAI/ML モデルをトレーニングし、入力データに最適な方法を学習することで、最も正確な結果を得ることができます。 |
| 最適化 | モデルの予測に基づいて自動的に最適化された予算配分を取得します。<br/>直感的なUIを使用して、数回クリックするだけで複数の予算シナリオを開発および比較できます。 |

{style="table-layout:auto"}


### 影響

この使用を適切に実装すると、次のような影響を与える可能性があります。

* 複数の目標と優先度をまたいでROIを最大化するマーケティング投資計画を作成します。
* 増分ROI曲線を使用して、予算配分を変更する機会を特定します。
* セルフサービスの民主化されたツールを使用して、月次または週次の予測を迅速に実施できます。

## Adobeアプリケーション全体で戦術的なインサイトを活用

このユースケースでは、顧客接点スコアにアクセスすることで、顧客セグメントとジャーニーに関する戦略的な増分インサイトを得ることができます。

+++ 詳細

### 課題

これらのユースケースアドレスの課題は次のとおりです。

* トップダウン型のソリューションだけでは、重要な最適化の機会を詳細に特定できない場合があります。
* 測定モデルの出力は膨大で記述が多く、インサイトやアクションにつながりにくいものです。
* モデルの透明性が低く、詳細なスコアも利用できないため、インサイトを得るための高度な分析を実施できません。


### アプローチ

このユースケースのステップベースのアプローチは次のとおりです。

| 手順 | 詳細 |
|---|---|
| モデル | AI/ML モデルを設定およびトレーニングして、一貫したMTA タッチポイントスコアとMMM集計結果を取得します。 |
| 分析 | 増分タッチポイント スコアをCustomer Journey Analyticsまたは外部BI ツールにエクスポートします。<br/>詳細な分析を実行し、タッチポイント スコアを使用して高度なダッシュボードを構築します。 |
| アクション | Real-Time Customer Data Platformを使用して、主要な消費者セグメントに類似したコンテンツを作成し、アクティベートします。<br/>今後のキャンペーンのために、顧客セグメント別にデータドリブン型マーケティング戦略を開発します。 |

{style="table-layout:auto"}


### 影響

この使用を適切に実装すると、次のような影響を与える可能性があります。

* 継続的な顧客行動とマーケティング戦術を把握し、マーケティングと組織全体の戦略的優先事項を把握する。
* 顧客や自社ビジネスに影響を与えるトレンドを迅速に検出し、ROIを最大化して戦略的対応を策定します。
